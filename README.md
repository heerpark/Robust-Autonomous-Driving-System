# 2. Object Detection: 데이터 중심의 성능 최적화

엣지 디바이스의 제한된 자원 환경에서는
모델을 대형화하는 것보다 데이터 품질 향상이 더 효과적이라는 점에 집중했습니다.

## 접근 전략

### a. 모델 규모의 한계 인지

* 실제 주행 테스트 결과
  모델 파라미터 증가보다 현실 환경 변수를 반영한 데이터가 성능에 더 큰 영향을 미침을 확인

### b. 양질의 데이터셋 직접 구축

* 약 2,900장 원본 이미지 수집
* 총 9개 클래스 (표지판, 신호등 등)
* 클래스당 약 300장 균형 라벨링

### c. 환경 적응형 데이터 증강

* 조도 변화
* 센서 노이즈
* 주행 중 흔들림

→ 데이터셋을 약 4,000장 규모로 확장

### d. 결과

작은 모델임에도 불구하고
실제 주행 환경에서 표지판을 높은 신뢰도로 안정적으로 탐지

# 3. 기본 제어 로직: 오프셋 기반 조향 제어

## a. 오프셋 적용

차량이 차선에 너무 붙지 않도록
목표 지점에 Offset을 부여하여 안전 거리를 유지합니다.

## b. 오차 계산 및 데드존 적용

화면 중심과 목표 지점의 거리를 계산하고,
미세 진동을 방지하기 위해 Dead Zone을 설정했습니다.

# 4. 문제 해결
## a. 객체 인식 오류 해결

영상 인식의 순간적인 오작동으로 인한
급제동 및 급회전을 방지하기 위해 다중 검증 필터링 시스템을 도입했습니다.

### 다중 검증 필터

* 객체 바운딩 박스 면적이 설정 범위 내에 존재
* 최근 5프레임 중 4프레임 이상 동일 객체 탐지 시에만 최종 판단

### 효과

* 멀리 있는 객체에 의한 조기 반응 방지
* 일시적 오인식 차단
* 판단 신뢰도 향상

## b. 차선 유실 시 이탈 방지

조명 변화 또는 가려짐으로 인해
차선 인식이 일시적으로 끊기는 문제를 해결하기 위해
회복 알고리즘을 설계했습니다.

### 1단계: 안정화 단계

* 차선 미검출 시 즉시 급조향하지 않음
* 일정 프레임 동안 저속 직진
* 환경 회복 대기

### 2단계: 차선 재탐색

* 미검출이 지속되면
* 이전에 추종하던 방향(Left/Right) 유지
* 천천히 해당 방향으로 기울이며 차선 재탐색
* 원래 경로로 복귀
